# Demo flow LoreForge

Сценарий рассчитан на 2-3 минуты и показывает LoreForge как рабочий demo-продукт: Roll20 + Notion для ГМа + плагины миров.

## Перед показом

```powershell
docker compose up -d
.\scripts\db\reset-dev-db.ps1
pnpm install
pnpm dev
```

Откройте:

```text
http://localhost:3000/login
```

## Сценарий показа

1. Открываем `/login`.
   Показываем dark SaaS стиль, glass login card, LoreForge слева и преимущества платформы.

2. Нажимаем `Войти`.
   Авторизация сейчас демо, кнопка ведёт на `/campaigns`.

3. Видим список кампаний.
   Открываем кампанию `Туман над Баклундом`.

4. На `/campaigns/1` показываем главный dashboard.
   За несколько секунд видно кампанию, активный плагин `Mistbound`, персонажей, NPC, локации, чат, расследование и последние события.

5. Показываем активный плагин Mistbound.
   Объясняем, что LoreForge поддерживает подключаемые миры и правила поверх общего ядра кампании.

6. Открываем `Персонажи`.
   Показываем путь, последовательность, духовность, рассудок, усвоение, риск потери контроля и способности персонажей.

7. Открываем `NPC`.
   Показываем dossier cards: публичное описание, теги и отдельный `Секрет ГМа`.

8. Открываем `Чат`.
   Показываем сообщения кампании и выделенные dice-сообщения.

9. Делаем быстрый бросок `d20`.
   Кнопка создаёт `dice_roll` через backend API и добавляет dice-сообщение в чат.

10. Показываем, что бросок появился в списке сообщений.
    Это демонстрирует связь чата, кубиков и кампании.

11. Открываем `Журнал`.
    Показываем timeline событий с публичным summary и отдельным секретным summary для мастера.

12. Завершаем формулировкой:

```text
LoreForge = Roll20 + Notion для ГМа + плагины миров.
```

## Что важно проговорить

- Roll20 в основном закрывает карту, токены и кубики.
- LoreForge ведёт всю структуру кампании: лор, NPC, локации, секреты, расследования, чат, сессии и правила.
- Плагины миров позволяют делать DnD-like, Mistbound-like, cyberpunk или авторские системы без переписывания ядра.
