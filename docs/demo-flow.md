# Demo flow LoreForge

Сценарий рассчитан на 2-3 минуты и показывает LoreForge как рабочий demo-продукт: Campaign OS для мастера RPG.

## Перед показом

```powershell
docker compose up -d
.\scripts\db\reset-dev-db.ps1
pnpm install
pnpm dev
```

Откройте:

```text
http://localhost:3000/login
```

## Сценарий показа на 2-3 минуты

1. Login.
   Открываем `/login`. Показываем dark SaaS стиль, glass login card, LoreForge слева и три преимущества платформы.

2. Campaigns.
   Нажимаем `Войти`. Авторизация сейчас демо, кнопка ведёт на `/campaigns`.

3. Dashboard.
   Открываем кампанию `Туман над Баклундом`. На `/campaigns/1` сразу видно кампанию, статистику, персонажей, NPC, локации, чат, расследование и последние события.

4. Mistbound plugin.
   Показываем активный плагин `Mistbound`. Объясняем, что LoreForge поддерживает подключаемые миры и правила поверх общего ядра кампании.

5. Characters.
   Открываем `Персонажи`. Показываем путь, последовательность, духовность, рассудок, усвоение, риск потери контроля и способности.

6. NPC SecretBlock.
   Открываем `NPC`. Показываем dossier cards: публичное описание, теги и отдельный `Секрет ГМа`.

7. Chat dice roll.
   Открываем `Чат`. Нажимаем быстрый бросок `1d20`. Кнопка создаёт `dice_roll` через backend API и добавляет dice-сообщение в чат.

8. Session log.
   Открываем `Журнал`. Показываем timeline событий с публичным summary и отдельным секретным summary для мастера.

9. Вывод.
   Завершаем формулировкой:

```text
LoreForge = Campaign OS для мастера RPG.
LoreForge = Roll20 + Notion для ГМа + плагины миров.
```

## Что важно проговорить

- Roll20 в основном закрывает карту, токены и кубики.
- LoreForge ведёт всю структуру кампании: лор, NPC, локации, секреты, расследования, чат, сессии и правила.
- Плагины миров позволяют делать DnD-like, Mistbound-like, cyberpunk или авторские системы без переписывания ядра.
